# 项目 - 协作规范

> 所有 Agent 必须严格遵守本规范。

---

## 沟通协作机制
**和其他角色进行沟通时，必须使用 @ 规范**：注意所有需要其他角色响应的沟通协作，都需要使用 @ 规范来进行。

@ 规范格式：
```
<at user_id="at_id">用户名称</at>
```

- `用户名称` 是必填的，必须填写群成员信息中对应的**准确名称**，系统会根据名称自动路由消息
- `at_id` 从群聊上下文的群成员信息（at_id映射表）中获取，如果能获取到请填写，获取不到可以留空
- **禁止写法**：`@用户名称`（纯文本，无效）

## 什么时候使用@规范
- **任务指派**：需要某角色立即执行任务时，必须 @ 该角色
- **请求确认**：需要某角色确认/审核时，必须 @ 该角色
- **任务完成**：任务完成后，告知任务完成情况，必须 @ 该角色，例如：Agent 完成任务后 @ 项目经理 角色汇报结果

**汇报格式示例**：
```
<at user_id="cli_xxx">项目经理</at> 用户模块 API 开发完成，包含注册、登录、信息管理 3 个接口，请指示下一步工作。
```
> at_id（`cli_xxx`）从群成员信息中获取，获取不到时可留空：`<at user_id="">项目经理</at>`

## 不能使用@规范的场景
- **信息同步**：仅告知信息、不需要对方执行动作时，不要 @ 该角色
**判断标准**：发送前问自己——"我需要这个人现在回复我/执行动作吗？"如果答案是"不需要"，就不要 @。


## **严格要求** 回复内容前需要对@规范进行检查
- 发送前自检消息体中如果汇报任务进展，分派新任务。需要使用@规范来寻求协作
- 回复的内容必须使用@规范来寻求协作，不能使用【已通过@规范通知项目经理】或者【已自动向项目经理汇报】这种文案。
- 严格区分"描述"和"执行"——@规范 是具体动作，不是文字描述
- **用户名称必须准确**，系统会根据名称路由消息；at_id 从群成员信息中获取，获取不到可留空
